## 0. 阅读须知（必读）

### 0.1 单位换算：格 → 像素

本游戏采用**格系统**：所有距离 / 速度 / 范围在 `DataRegistry` 中以「格」为单位存储，运行时通过 `BattleConstants.px()` 转为像素。

```
1 格 = CELL_SIZE = 20 像素（游戏空间）
屏幕上 1 格的实际高度 = CELL_SIZE × Y_COMPRESS = 20 × 0.7863 ≈ 15.73 像素
```

| 换算 | 公式 | 示例 |
|------|------|------|
| 格 → 游戏空间像素 | `格数 × 20` | `move_speed 1.0 格/秒 = 20 px/s` |
| 格 → 屏幕高度像素 | `格数 × 15.73` | `2 格射程 ≈ 31.5 屏幕像素` |

> **重要**：本文档所有数值均以**格**为单位（与 `DataRegistry` 一致）。括号内会标注关键换算结果。

### 0.2 移速档位对照

《皇室战争》的移速档位换算为本项目的「格/秒」数值（`move_speed` 字段）：

| 显示移速 | 原版内部数值 | 本项目 `move_speed`（格/秒） | 游戏空间速度（px/s） |
|----------|--------------|------------------------------|----------------------|
| 慢速 Slow | 45 | **0.75** | 15 |
| 中速 Medium | 60 | **1.0** | 20 |
| 快速 Fast | 90 | **1.5** | 30 |
| 极快 Very Fast | 120 | **2.0** | 40 |
| 不可移动（建筑） | — | **0.0** | 0 |

## 1. 字段字典（理解每个数值的含义）

> 这一节解释每个字段的含义、取值范围和对游戏行为的影响。后续数据表不再重复解释。

### 1.1 实体基础字段（单位 + 塔通用）

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| `id` | String | — | 唯一标识，代码中用作查询 key |
| `display_name` | String | — | 中文显示名 |
| `max_hp` | int | — | 最大生命值。归零即死亡 |
| `shield` | int | — | 护盾上限。受伤时优先扣护盾，护盾不溢出到 HP。`0` = 无护盾 |

### 1.2 移动字段（仅单位）

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| `move_speed` | float | 格/秒 | 移动速度。`0.0` = 不可移动（建筑/迫击炮）。换算见 §0.2 |
| `movement_type` | String | — | `"ground"`（地面）/ `"air"`（飞行）。飞行单位自动获得 `altitude=2.5` 离地高度（纯视觉），地面攻击无法锁定它 |
| `can_jump_river` | bool | — | 可选。`true` = 地面单位可跳河（比较走桥/跳河两条路线，跳河更短才跳）。当前 `hog_rider`、`prince` 开启 |
| `sight_range` | float | 格 | 视野范围。无攻击目标时，在此范围内发现敌人会转向追击 |
| `movement_targeting` | String | — | `"any"`（自由索敌，可被任何敌方单位拉扯）/ `"building_only"`（只认塔和建筑，不会偏离主路线） |

### 1.3 碰撞与物理字段（★ 用户重点关注）

本游戏的碰撞是**轻量位置分离**（非物理引擎），由 `CollisionSystem`（静态工具）每帧迭代处理。以下字段直接决定碰撞行为：

| 字段 | 类型 | 单位 | 说明 | 影响的游戏行为 |
|------|------|------|------|----------------|
| `collision_radius` | float | 格 | **碰撞体半径**（圆形碰撞）。决定两个实体之间要保持的最小距离（两圆相切距离 = 半径之和） | 实体间互推位置分离；部署时单位之间不重叠；过桥时排队间距 |
| `hurt_radius` | float | 格 | **受击半径**（圆形）。法术 / 溅射 / 投射物命中判定用的范围 | 一个法术/炮弹爆炸时，中心点到此距离内的实体会被波及 |
| `mass` | int | — | **碰撞质量**。越大被推得越少 | `CollisionSystem` 按质量反比推挤（重单位推不动，轻单位被推开）。**`0` = 完全不可移动**（塔、迫击炮），且自动成为其他单位的寻路障碍 |
| `shadow_size` | float | 格 | **影子椭圆水平半径**（可选，未配置时退化为 `collision_radius`） | 仅视觉。`UnitBase._draw()` 绘制半透明黑色椭圆，Y 轴压缩 `SHADOW_SQUASH=0.35` 变扁。地面单位 alpha 0.28，飞行单位 alpha 0.18 |

**质量 `mass` 取值参考**：

| mass 值 | 含义 | 典型单位 |
|---------|------|----------|
| 0 | 不可移动，自动成为寻路障碍 | 塔、迫击炮 |
| 3 | 轻量（易被推开） | 哥布林、弓箭手、重甲亡灵 |
| 4 | 较轻 | 野猪骑士、小皮卡 |
| 5 | 中等 | 火枪手 |
| 6 | 较重 | 骑士、气球兵 |
| 7 | 重 | 王子 |
| 10 | 极重（几乎推不动） | 巨人 |

**碰撞半径 `collision_radius` 直观大小**：

| 半径（格） | 直径（格） | 游戏空间直径（px） | 典型单位 |
|-----------|-----------|-------------------|----------|
| 0.3 | 0.6 | 12 | 气球兵 |
| 0.45 | 0.9 | 18 | 小皮卡 |
| 0.5 | 1.0 | 20 | 骑士、火枪手、弓箭手、哥布林 |
| 0.6 | 1.2 | 24 | 野猪骑士、迫击炮、重甲亡灵 |
| 0.65 | 1.3 | 26 | 王子 |
| 0.75 | 1.5 | 30 | 巨人 |
| 1.5 | 3.0 | 60 | 公主塔（3×3 格） |
| 2.0 | 4.0 | 80 | 国王塔（4×4 格） |

### 1.4 攻击字段（`attacks` 数组内每一项）

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| `name` | String | — | 攻击名称（如 `sword`、`musket_shot`），仅标识 |
| `targeting` | String | — | `"any"`（任意敌方）/ `"building_only"`（只打塔和建筑） |
| `attack_ground` | bool | — | 能否攻击地面单位 |
| `attack_air` | bool | — | 能否攻击空中单位 |
| `attack_range` | float | 格 | 攻击射程。进入此距离即停下攻击 |
| `min_attack_range` | float | 格 | 可选。**最小射程（近距离盲区）**。索敌时排除比此距离更近的目标。当前仅迫击炮使用 |
| `attack_interval` | float | 秒 | 攻击间隔（两次攻击之间的冷却） |
| `first_attack_delay` | float | 秒 | 起手延迟（部署/锁定目标后第一次攻击的等待时间，俗称"起手速度"） |
| `damage_delay` | float | 秒 | 可选。**出手到造成伤害的延迟**，用于对齐攻击动画关键帧（如挥剑劈下的那一帧） |
| `delivery` | String | — | `"instant"`（即时命中，近战）/ `"projectile"`（发射飞行物，远程） |
| `trajectory` | String | — | 飞行物弹道：`"linear"`（直线）/ `"ballistic"`（高抛抛物线）/ `"homing"`（追踪）。仅 `delivery=projectile` 时有效 |
| `impact_type` | String | — | `"single"`（单体）/ `"splash"`（范围溅射） |
| `impact_radius` | float | 格 | 溅射半径。仅 `impact_type=splash` 时有效 |
| `damage` | int | — | 单次伤害值 |
| `projectile_speed` | float | 格/秒 | 飞行物速度。仅 `delivery=projectile` 时有效 |
| `arc_height` | float | 格 | 可选。最大射程处的抛物线弧高，近处按距离比例自动降低。仅 `trajectory=ballistic` 时有效 |
| `ramp_damage` | Array | — | 可选。**递增伤害阶段值**（如 `[43, 158, 847]`）。配置此项即启用递增伤害机制（地狱塔光束）。与 `ramp_thresholds` 一一对应 |
| `ramp_thresholds` | Array | 秒 | 可选。**各阶段锁定时间阈值**（升序）。持续锁定同一目标达阈值后切换到对应阶段伤害；目标切换/离开射程时重置累计 |

### 1.5 特殊机制字段

| 字段 | 适用 | 说明 |
|------|------|------|
| `deploy_time` | **所有单位** | 部署时间（秒）。部署期间（`is_deployed=false`）不能索敌/攻击/移动，但可受伤。普通单位 1.0 秒，mortar 3.5 秒，地狱塔 1.0 秒。0.18.0 起所有单位均配置此字段 |
| `lifespan` / `lifespan_damage_per_sec` | 地狱塔、迫击炮（建筑） | 建筑寿命：`lifespan` 寿命到期自毁（普通单位 = 0 无寿命）；`lifespan_damage_per_sec` 自然掉血速率（HP/秒，不填则按 `max_hp/lifespan` 计算，使寿命到时血量恰好归零） |
| `ramp_damage` / `ramp_thresholds` | 地狱塔 | 递增光束：攻击配置（`attacks[]` 内），见 §1.4。持续锁定同一目标伤害按阶段递增 |
| `beam_emit_offset_y` | 地狱塔 | 光束发射点 Y 偏移（像素，负=上移）。塔顶喷口位置，由 `InfernoBeam` 组件读取绘制光束起点 |
| `death_damage` / `death_radius` / `death_fuse_time` | 气球兵 | 死亡炸弹：单位死亡时发出 `death_damage_triggered` 信号，由 EffectManager 生成 `DelayedDamageEffect`，引信到期后造成范围伤害 |
| `charge.min_charge_distance` / `charge_move_speed` / `charge_damage` | 王子 | 冲锋机制：持续直线移动达到此距离后进入冲锋，移速提升、命中伤害变为 `charge_damage`。攻击出手或受击时退出冲锋 |
| `tower_type` | 塔 | `"king"`（国王塔）/ `"guard"`（公主塔） |
| `sprite.player_texture` / `enemy_texture` | 塔 | 队伍差异化 PNG 贴图路径 |
| `sprite.visual_scale` / `visual_offset_y` | 塔 | 精灵缩放与垂直微调（底部对齐由代码自动计算，Y_COMPRESS 反向补偿） |

### 1.6 视觉动画字段（`animation` 子字典，可选）

无 `animation` 字段时退化为 ColorRect 占位方块（开发阶段可见）。有此字段时由 `SpriteAnimator` 创建 AnimatedSprite2D 渲染。

| 字段 | 单位 | 说明 |
|------|------|------|
| `visual_offset_x` / `visual_offset_y` | 像素 | 精灵水平/垂直偏移（负=上移）。目测微调用 |
| `visual_scale` | — | 缩放系数（如 1254px 大图缩到 99px ≈ 0.079） |
| `health_bar_y` | 像素 | 血条 Y 偏移（负=上移） |
| `texture_filter` | — | `"nearest"`（像素风）/ `"linear"`（高清图） |
| `jump_frame` | — | 可选。跳河期间锁定显示第 N 帧（0-indexed），仅野猪骑士 |
| `states` | — | 各动画状态配置，键名支持方向后缀（`walk_front` / `attack_back` 等），SpriteAnimator 先匹配全名再回退基础名 |

### 1.7 音效字段（`sfx` 子字典，可选）

键为触发时机，值为 `sound_data` 中的事件 id（见 DataRegistry）。

| 键 | 触发时机 |
|----|----------|
| `deploy` | 单位部署时 |
| `attack` | 单位攻击出手时 |
| `move` | 单位移动时（循环，仅野猪骑士） |
| `charge` | 王子进入冲锋时 |
