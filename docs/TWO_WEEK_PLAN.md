# 两周开发计划

## 概述

本项目是一个课程作业，开发周期两周。目标是在两周内完成一个可运行、可演示的 2D 像素风卡牌塔防对战原型。

---

## 第 1 周：核心系统

### 第 1 天：理解项目

**目标**：能打开项目，理解基本结构

**任务**：
- [ ] 阅读 `docs/PROJECT_OVERVIEW.md`
- [ ] 阅读 `docs/GODOT_LEARNING_NOTES.md`
- [ ] 用 Godot 编辑器打开项目
- [ ] 运行项目，观察主菜单和战斗场景
- [ ] 尝试 K 生成单位，观察单位行为

**交付**：项目能启动，能看到主菜单和战斗场景

### 第 2 天：战场和坐标

**目标**：理解战场布局

**任务**：
- [ ] 打开 `Arena.tscn`，观察各个区域的坐标
- [ ] 打开 `BattleConstants.gd`，理解塔的位置定义
- [ ] 在编辑器中拖动塔的位置，观察效果
- [ ] 理解 Y 轴方向（上=敌方，下=玩家）

**交付**：理解战场坐标系统

### 第 3 天：塔系统

**目标**：理解塔的行为

**任务**：
- [ ] 打开 `TowerBase.gd`，阅读 `setup()`、`_process()`、`find_target()`
- [ ] 修改 `DataRegistry.gd` 中塔的血量，观察变化
- [ ] 用 K 生成单位靠近塔，观察塔攻击单位
- [ ] 尝试修改塔的攻击范围和伤害

**交付**：理解塔的攻击机制

### 第 4 天：单位系统

**目标**：理解单位的行为

**任务**：
- [ ] 打开 `UnitBase.gd`，阅读完整代码
- [ ] 理解单位的三种行为状态：移动、追击、攻击
- [ ] 修改单位的移动速度，观察差异
- [ ] 在 `DataRegistry.gd` 中添加新单位类型

**交付**：理解单位的移动和攻击逻辑

### 第 5 天：卡牌 UI（待实现）

**目标**：添加底部卡牌栏

**任务**：
- [ ] 创建 `CardSlot.tscn` 和 `CardSlot.gd`
- [ ] 创建 `CardBar.tscn` 和 `CardBar.gd`
- [ ] 在 HUD 中显示 4 张卡牌
- [ ] 实现点击卡牌选择、再点击战场部署

**交付**：玩家能通过卡牌 UI 部署单位

### 第 6 天：能量 UI 和卡牌部署

**目标**：完善能量和部署交互

**任务**：
- [ ] 创建 `EnergyBar.tscn`
- [ ] 卡牌费用不足时变暗
- [ ] 部署区域外点击时提示
- [ ] 测试完整出牌流程

**交付**：玩家能正常出牌

### 第 7 天：敌方 AI 调试

**目标**：确保敌方 AI 正常工作

**任务**：
- [ ] 观察敌方 AI 出牌频率
- [ ] 调整 AI 出牌间隔
- [ ] 测试长时间运行的稳定性

**交付**：战斗能自然进行

- [ ] 优化 HUD 布局
